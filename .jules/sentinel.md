## 2024-06-27 - [Add basic JSON string sanitization]
**Vulnerability:** Graph snapshots parsed from JSON could contain XSS vectors (e.g., `javascript:` URIs, inline scripts, CSS expressions) in strings like tags, attributes, or graph identifiers. Since we intend to allow users to control explicit styling (like colors) via JSON payloads in the future, these strings need to be safely parsed.
**Learning:** Preventing prototype pollution via `__proto__` when using modern JavaScript spread syntax (`{...obj}`) natively ignores prototype setting, allowing us to safely `Object.freeze` without elaborate deep-copy checks. However, regex-based sanitization is still necessary to cleanse XSS patterns from strings embedded in the graph schema payload before it touches the DOM via GraphView renderer content rendering.
**Prevention:** Ensured `assertNonEmptyString` enforces strict checking against a unified `sanitizeString` logic. Applied this sanitization to parsed structural JSON fields directly inside `createGraphSnapshot`, `freezeTags`, and `freezeAttributes` prior to freezing them into immutable models.
