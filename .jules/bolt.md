## 2026-06-28 - Prevent Maximum call stack size exceeded with Math.max\n**Learning:** Using the spread operator with `Math.max(...iterable)` can throw a `RangeError: Maximum call stack size exceeded` when dealing with very large datasets (like large graphs). This happens because engines place a hard limit on the number of arguments a function can take.\n**Action:** Always prefer standard `for...of` loops for iterating over large sets to find max/min values instead of spreading into function arguments.
